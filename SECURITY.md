# Security Policy

This is a build-in-public project at version 0.2.0. It is not production
software and the README says so, but security reports are still welcome
and taken seriously.

## Reporting a vulnerability

Please do not open a public issue for security problems. Report it
privately through GitHub: open the Security tab and choose "Report a
vulnerability". The details stay private until a fix is available.

Please include:

- What the problem is and why it is a security issue
- Steps to reproduce, ideally a request against a running server
- The commit you tested (`git rev-parse --short HEAD` is enough)

You will get an acknowledgement, and updates as the fix lands.

## Supported versions

The project is pre-1.0 and moves quickly. Only the current `main` branch
is supported. Please reproduce against the latest commit before reporting.
